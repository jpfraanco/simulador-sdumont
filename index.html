Simulador mínimo — dashboard + terminal funcionais. Narrator/tour vem depois.

Como usar agora

  1. Olha o dashboard à direita: são os 377 nós do SDumont v1, agrupados por tipo (CPU, BIGMEM, GPU, GDL). Cores: verde=livre, amarelo=parcial, vermelho=cheio.
  2. Abaixo do grid, a fila de jobs mostra outros usuários fictícios rodando workloads reais do LNCC (GROMACS, PyTorch, OpenFOAM, etc).
  3. No terminal (canto inferior direito), você começa na "sua máquina local". Digite ssh unseen@sdumont15 para entrar no cluster.
  4. Depois do SSH, tente: ls, cat train_palmvein.srm, module avail, squeue, sinfo, sbatch train_palmvein.srm, squeue --me.
  5. Veja o job aparecer na fila à direita e o nó ficar vermelho quando ele começar a rodar.

Comandos principais

  • ls, cd, pwd, cat <arquivo>
  • ssh unseen@sdumont15, exit
  • module avail, module load cuda/11.2_sequana
  • sbatch <script.srm>
  • squeue, squeue --me
  • sinfo, sinfo -p gdl
  • scancel <JOBID>
  • nvidia-smi (só quando em nó GPU)
  • help — lista tudo

Cluster avança a cada 2s. Outros usuários submetem jobs automaticamente.
